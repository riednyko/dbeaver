Frequently Asked Questions about DBeaver

This is draft version of "Frequently Asked Questions", more information will be added soon

General

Why "DBeaver"

DBeaver named in contrast to SQuirreL, popular java-based database client. It's something like "database beaver versus SQL squirrel". DBeaver is pronounced as dee-beaver.

What is "JKISS"

It's our company name. It's an acronym ("Just Keep It Straight and Simple" or "Just Keep It Simple, Stupid"). JKISS is pronounced as jey-kiss.

Who makes it and for what reason

Check Team page to see who we are.

For now there is no any certain goal of this project. We just want to make a good (better than ony other) universal tool to work with databases.

This is a completely non-commercial project, nobody pays us for the job, there are no donations or any other kind of funding. This project is powered by our pleasure for 100%.

So far.

May I access DBeaver source code?

Not yet. At the moment DBeaver is completely freeware but it is not opensource. It may become so in the future. Actually we have plans to open sources this year but first we want to make really fully functional and stable version.

Installing and running

DBeaver doesn't start

A1: You have old Java or not Java VM installed. DBeaver require Java 1.6. You may get newest Java from http://java.com or you may use automatic installer (Windows version) which will check and install Java 1.6 on demand. If you have different versions of JVM on your computer then you may specify particular one in command line (check readme file for command line arguments description).

A2: You use wrong DBeaver version (e.g. 64bit on 32bit OS)

A3: You have extracted archive with newer version of DBeaver in the folder where older version has been installed. Sometimes it may cause file versions' conflicts. You may fix it by removing all contents of folder "configuration" excepting the file "config.ini".

Where DBeaver stores it's configuration and how can I change it

By default DBeaver stores all its data in user's home ".dbeaver" folder. E.g. "C:\Users\<UserName>\.dbeaver" or "/home/<UserName>/.dbeaver". This folder won't be deleted upon uninstall or reinstall so new version of DBeaver will use existing configuration/data. You may freely replace this folder - DBeaver will just start with default settings.

You may change default configuration location using command line parameter "-data". Check readme.txt file for command line parameters description.

Database driver not found

If you use custom driver then check driver's properties in driver manager. Make sure that all driver's libraries (jars) are valid and that driver's class could be found by DBeaver (use "Find Class" button). If DBeaver can't locate any driver class in specified libraries then likely you have invalid or broken libraries.
